# 範囲外和暦のフォールバック修正と漢数字キャッシュ実装計画

**目的**：和暦として認識した範囲外日付を標準パーサーへ渡さず、`Wareki::UnsupportedDateRange` を保持する。
同時に、日付と時刻の漢数字フォーマットで繰り返される `YaKansuji.to_kan` を小整数キャッシュへ置き換える。

**対象**：包括レビューの所見1と所見2だけを修正する。
負の年の漢数字表現、通常の `strftime` と `Date.parse` の高速経路、暦テーブルの遅延ロードは対象外とする。

**互換性条件**：Ruby 2.0で解釈できる構文だけを使う。
既存の寛容な標準パーサーへのフォールバックと、`Wareki::InvalidDate` を送出する挙動は維持する。

**実施状態**：2026-07-14に実装と独立レビューを完了した。
最終検証はRSpec 87件成功、RuboCop 17ファイル違反なし、Ruby構文検査成功である。
キャッシュ初期化には約0.95 msと約9,400個の一時割り当てが増えるが、定常処理では `%JF` が約30.8 µsから2.1 µs、`%JTF` が約32.1 µsから2.5 µsへ短縮した。

## Task 1：範囲外和暦を例外として保持する

**難易度**：中。
例外の種類によってフォールバック可否を分ける必要があり、`Wareki.parse_to_date` とモンキーパッチした `Date.parse` の整合性を確認する。

**変更対象**：

- `lib/wareki/common.rb`
- `lib/wareki/std_ext.rb`
- `spec/wareki_spec.rb`
- `spec/std_ext_spec.rb`

- [ ] `Wareki.parse_to_date('皇紀100年5月4日')` が `Wareki::UnsupportedDateRange` を送出する失敗テストを追加する。
- [ ] `Date.parse('皇紀100年5月4日')` が同じ例外を送出する失敗テストを追加する。
- [ ] `Date._parse` と `Time.parse` も同じ入力で範囲外例外を保持することを回帰テストに含める。
- [ ] `parse_to_date` と `Date.parse` の rescue 対象から `UnsupportedDateRange` を外す。
- [ ] ISO日付、和暦でない日本語日付、構文不一致文字列が従来どおり標準パーサーへフォールバックすることを確認する。
- [ ] `Wareki::InvalidDate` が従来どおりフォールバックされないことを確認する。
- [ ] 対象specと全specを実行する。

**完了条件**：認識済みの範囲外和暦は全入口で `UnsupportedDateRange` になり、和暦として認識できない入力だけが標準パーサーへ渡る。

## Task 2：小整数の漢数字変換をキャッシュする

**難易度**：高。
日付と時刻の全フォーマット経路を共通化し、出力互換性と起動時コストを保ちながら性能を測定する。

**変更対象**：

- `lib/wareki/utils.rb`
- `lib/wareki/date.rb`
- `spec/utils_spec.rb`
- `spec/date_spec.rb`
- `spec/std_ext_spec.rb`

- [ ] 変更前の `%JF` と `%JTF` について、1呼び出し当たりの時間と割り当て数を記録する。
- [ ] `0..99` の `:simple` 漢数字を凍結配列として一度だけ生成する。
- [ ] キャッシュ範囲内は配列参照、範囲外は `YaKansuji.to_kan(num, :simple)` を使う共通ヘルパーを追加する。
- [ ] 日付の `%JGk`、`%JGK`、`%JOk`、`%JIk`、`%JSk`、`%JDk`、`%JDK` を共通ヘルパーへ切り替える。
- [ ] 時刻の `%JTF`、`%JTHk`、`%JTMk`、`%JTSk` を共通ヘルパーへ切り替える。
- [ ] 0、1、9、10、31、59、60、99が従来と同じ文字列になるテストを追加する。
- [ ] 100以上と負数が従来どおり依存gemへ委譲されるテストを追加する。
- [ ] `%JF` と `%JTF` の既存出力が変わらないことを回帰テストで確認する。
- [ ] 変更後に同じベンチマークを実行し、性能と割り当て数を比較する。
- [ ] 対象spec、全spec、RuboCopを実行する。

**完了条件**：既存出力を維持し、代表入力の `%JF` と `%JTF` で `YaKansuji.to_kan` の反復呼び出しを除去する。
ベンチマークでは、変更前より時間と割り当て数の両方が減少していることを確認する。
